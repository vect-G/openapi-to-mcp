# openapi-to-mcp v0.3.0

Release date: 2026-03-18

## TL;DR

`openapi-to-mcp` now supports real-world OpenAPI complexity better:

- External `$ref` (local files and remote URLs)
- Nested `$ref` expansion inside referenced files
- Swagger 2 `in: body` request mapping

If your API specs are split across files or still in Swagger 2, this release is for you.

## What's new

### 1) External `$ref` support

You can now load specs that reference schemas in other files or URLs.

Examples:

- `./schemas.yaml#/OrderCreate`
- `https://example.com/schemas.json#/components/schemas/User`

### 2) Nested `$ref` expansion

Referenced external documents can contain their own internal `$ref`, and those are expanded correctly.

### 3) Swagger 2 body parameter compatibility

Legacy specs that define request body as:

- `in: body`
- with `consumes`

are now converted into MCP `body` input automatically.

## Example commands

```bash
openapi-to-mcp list examples/external-ref/api.yaml --json
openapi-to-mcp list examples/swagger2.yaml --json
```

## Stability

- Test suite: `12 passed`
- Existing CLI flows remain compatible (`run`, `list`, `mcp-config`, `init-config`)


```bash
python -m pip install -e .[dev]
```
